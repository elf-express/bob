# 專案結構功能說明與連動檔案地圖

## 一、功能主要實現
- 在左側「專案結構」頁顯示目前綁定根目錄的檔案樹
- 支援搜尋、展開/收合、右鍵操作（新增/刪除/重新命名）
- 支援檔案標註（Annotation / Purpose / Relations / User Note）
- 支援標籤顯示與自訂標籤
- 支援掃描快照（scripts 掃描 + tree 掃描）與掃描狀態顯示

## 二、對應需求面向
- 新手可快速理解專案結構與檔案用途
- 改某個檔案前，可先看到關聯描述與影響範圍
- 讓 AI 有可寫回的結構化資料（標註、筆記、標籤）
- 安全邊界：只允許寫入工具資料（DB），不允許修改主專案檔案系統

## 三、為什麼做（目的）
- 降低閱讀成本：從「看檔名」升級為「看意圖與關聯」
- 降低修改風險：先看關聯再改碼
- 提升 AI 協作品質：AI 先讀註解/關聯再改，降低誤改

## 四、連動檔案地圖（重點）

### 1) 頁面入口與版位
- `.bob-tools/public/index.html`
- `.bob-tools/public/js/modern-app.js`（切換到 `project-explorer` view）

### 2) 專案樹 UI 與互動核心
- `.bob-tools/public/js/file-tree.js`
  - 載入樹、搜尋框、展開收合、右鍵選單、scan 按鈕
- `.bob-tools/public/js/file-tree-icons.js`
  - 檔案/資料夾 icon 對應
- `.bob-tools/public/js/file-tree-tags.js`
  - tag UI 工具
- `.bob-tools/public/js/file-tree-annotation.js`
  - 右側標註/描述/關聯/使用者筆記面板
- `.bob-tools/public/css/modern-ui.css`
  - 若改全域樣式或共用元件，需同步檢查

### 3) 前端 API 呼叫封裝
- `.bob-tools/public/js/file-tree-api.js`
  - `fetchFiles / searchFilesApi / fetchProjectStats`
  - `scanCommandsApi / scanTreeApi / fetchLatestScanSummary`
  - `fetchProjectBindingStatus`

### 4) 後端路由層
- `.bob-tools/src/routes/index.js`（掛載路由）
- `.bob-tools/src/routes/files.js`
  - `/api/files`、`/api/files/search`、`/api/files/stats`
  - `/api/files/metadata`、`/api/files/annotate`
  - `/api/files/create`、`/api/files/delete`、`/api/files/rename`（唯讀模式下回傳 `403 FS_WRITE_DISABLED`）
- `.bob-tools/src/routes/scan.js`
  - `/api/scan/commands`、`/api/scan/tree`、`/api/scan/latest`
- `.bob-tools/src/routes/project.js`
  - 綁定狀態（專案根目錄資訊）

### 5) 後端服務層
- `.bob-tools/src/services/file-system.js`
  - 檔案列舉、搜尋、統計、路徑安全、防止 traversal
  - tree snapshot 產生與忽略目錄規則
- `.bob-tools/src/services/annotation-service.js`
  - metadata 讀寫整合（annotation/purpose/relations/userNote）
- `.bob-tools/src/services/tag-service.js`
  - 標籤規則與自訂標籤

### 6) DB/Repository
- `.bob-tools/src/db/repositories/annotation-repository.js`
  - `file_annotations` 的 CRUD
- `.bob-tools/src/db/repositories/scan-repository.js`
  - 掃描執行記錄與快照（`commands_snapshot` / `tree_snapshot` / `scan_runs`）
- `.bob-tools/src/db/migrations/*.sql`
  - 若新增欄位或資料結構，需補 migration

### 7) 多語系
- `.bob-tools/public/locales/zh-TW/common.json`
- `.bob-tools/public/locales/zh-CN/common.json`
- `.bob-tools/public/locales/en-US/common.json`

## 五、常見修改情境與必看連動

### 情境 A：只改搜尋框 UI
- 必看：`file-tree.js`、`modern-ui.css`
- 可能連動：`index.html`（容器結構）

### 情境 B：新增右鍵選單功能
- 必看：`file-tree.js`（context menu 建構與事件）
- 可能連動：`files.js`（新增 API）+ `file-system.js`（實作）
- 注意：不得繞過唯讀策略去修改主專案檔案

### 情境 C：擴充標註欄位
- 必看：`file-tree-annotation.js`、`files.js`、`annotation-service.js`、`annotation-repository.js`
- 必做：DB migration + 三語系文案

### 情境 D：調整掃描邏輯
- 必看：`scan.js`、`file-system.js`、`scan-repository.js`
- 可能連動：`file-tree-api.js`、`file-tree.js`（scan 結果顯示）

## 六、給 AI 修改代碼的最小指令模板
- 先閱讀：`index.html`、`file-tree.js`、`file-tree-api.js`、`routes/files.js`
- 再判斷是否涉及：`file-system.js`、`annotation-service.js`、`scan.js`
- 若新增欄位：同步更新 migration、repository、前端 UI、i18n
- 最後驗證：功能操作 + API 回傳 + i18n 不出 raw key
